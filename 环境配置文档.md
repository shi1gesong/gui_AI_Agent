# 开发环境配置文档

项目：基于多模态大模型的桌面GUI智能体开发与优化
环境基础：WSL2（D盘） + Windows原生，物理机为 RTX 5070 Ti
更新说明：v1版本中部分依赖版本号（尤其torch）经实际搭建验证不可用，本版本为实测跑通后的最终配置。

## 0. 重要前提：WSL2磁盘位置检查

**在开始之前务必先检查C盘剩余空间**（`df -h` 或 Windows文件资源管理器查看C盘属性）。若C盘可用空间低于20G，WSL2虚拟磁盘容易在安装大型库（如torch）时出现 `Bus error`（本质是磁盘空间不足导致内存映射mmap失败）。

若空间紧张，建议提前把WSL发行版迁移到空间充足的盘（如D盘），在**Windows PowerShell**（不是WSL内部）执行：

```powershell
mkdir D:\wsl-backup
mkdir D:\wsl
wsl --shutdown
wsl --export Ubuntu D:\wsl-backup\ubuntu.tar
wsl --unregister Ubuntu
wsl --import Ubuntu D:\wsl\ubuntu D:\wsl-backup\ubuntu.tar --version 2
```

⚠️ **务必确认 `--export` 那一步真正执行成功（能看到"正在导出"进度提示且无报错）之后，再执行 `--unregister`**。顺序搞反或者目标文件夹没提前建好，会导致原WSL环境内容清空且无法恢复。

## 1. 环境隔离策略

GUI Agent项目与其他项目（如medical-rag）分开建conda环境，且**Windows原生端和WSL2端也是两个完全独立的环境**，二者通过文件系统交换数据：

- **WSL2 端（`gui-agent`）**：负责大模型推理、Agent框架、LoRA微调（用得到GPU的部分）
- **Windows原生端（`gui-agent-win`）**：负责屏幕截图、OCR、鼠标键盘控制（WSL2访问不了Windows图形界面，这部分必须在Windows原生Python里跑）

```bash
# WSL2 内执行
conda create -n gui-agent python=3.10 -y
conda activate gui-agent
```

若conda提示需要接受渠道服务条款（较新版本conda会遇到），先执行：
```bash
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/main
conda tos accept --override-channels --channel https://repo.anaconda.com/pkgs/r
```

## 2. PyTorch + CUDA（WSL2端，实测可用版本）

⚠️ **RTX 5070 Ti 是较新的Blackwell架构（CUDA算力sm_120），torch 2.2.x/cu121 等较旧版本不支持该显卡**，安装后会提示 `CUDA capability sm_120 is not compatible`。经实测，以下版本可正常识别GPU：

```bash
pip install torch==2.7.0 torchvision torchaudio --index-url https://download.pytorch.org/whl/cu128
```

验证：

```bash
python -c "import torch; print(torch.__version__, torch.cuda.is_available(), torch.cuda.get_device_name(0))"
# 预期输出：2.7.0+cu128 True NVIDIA GeForce RTX 5070 Ti
```

## 3. Agent框架与大模型调用依赖（WSL2端）

```bash
pip install langchain langchain-community llama-index
pip install transformers accelerate peft bitsandbytes datasets
```

⚠️ **已知问题**：安装上述包后，`torchaudio` 可能与 `torch` 版本出现 ABI 不兼容，import transformers/peft 时报 `undefined symbol: torch_library_impl`。本项目不涉及音频处理，直接卸载即可：

```bash
pip uninstall torchaudio -y
```

若出现 numpy 相关的 `_ARRAY_API not found` 警告（NumPy 2.x与部分预编译模块不兼容），可选择性降级：

```bash
pip install "numpy<2"
```

验证整体环境：

```bash
python -c "import langchain; import transformers; import peft; print('all imports ok')"
```

## 4. 桌面感知与控制依赖（Windows原生端，独立环境）

在 **Anaconda Prompt**（不是WSL终端）中执行：

```bash
conda create -n gui-agent-win python=3.10 -y
conda activate gui-agent-win
pip install pyautogui mss pynput opencv-python paddleocr easyocr pillow
```

验证截图功能：

```bash
python -c "import pyautogui; img = pyautogui.screenshot(); img.save('test_screenshot.png'); print('截图成功，尺寸:', img.size)"
```

验证OCR功能（首次运行会自动下载识别模型，需联网，耐心等待1-2分钟）：

```bash
python -c "import easyocr; reader = easyocr.Reader(['ch_sim','en']); result = reader.readtext('test_screenshot.png'); print('识别到的文字数量:', len(result))"
```

> PaddleOCR 留到第2周正式开发感知模块时再装并与EasyOCR做精度/速度对比，本周先用EasyOCR完成环境验证。

## 6. 版本控制

```bash
mkdir -p ~/gui-desktop-agent && cd ~/gui-desktop-agent
git init
mkdir perception control agent models data eval reports demo
```

目录结构：

```
gui-desktop-agent/
├── perception/       # 截图、OCR、UI元素识别（Windows端运行）
├── control/          # 鼠标键盘控制（Windows端运行）
├── agent/            # LangChain Agent 框架、任务规划
├── models/           # 微调脚本、LoRA权重
├── data/              # 数据集处理脚本与缓存
├── eval/              # 评测脚本与报告
├── reports/           # 每周实验报告
└── demo/               # 演示视频、截图素材
```
