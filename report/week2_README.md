# 第2周实验报告

项目：基于多模态大模型的桌面GUI智能体开发与优化
第2周：桌面感知与控制核心模块开发 | 2026年7月22日

## 一、本周任务目标

按项目周计划，第2周核心任务为：（1）实现跨平台屏幕截图，支持多分辨率适配；（2）集成OCR工具，实现屏幕文字与UI元素识别；（3）开发鼠标键盘控制模块，支持点击、输入、滚动、拖拽等基本操作；（4）实现UI元素坐标定位与边界框绘制功能。全部代码在Windows原生环境（gui-agent-win）中开发与验证，因WSL2无法访问Windows图形界面。

## 二、模块开发与测试结果

### 2.1 屏幕截图模块（perception/screenshot.py）

基于mss库实现全屏与指定区域截图，支持多显示器场景（通过monitor索引区分）。实测检测到1个物理显示器，分辨率2560×1440，全屏与区域截图均验证通过，输出尺寸与预期完全一致。

### 2.2 OCR文字识别模块（perception/ocr_detector.py）

实现了统一接口封装EasyOCR与PaddleOCR两种引擎，输出格式统一为`[文字, 边界框坐标, 置信度]`列表，并支持在原图上可视化绘制识别出的边界框。两引擎在同一张测试截图上的实测对比如下：

| 指标 | EasyOCR | PaddleOCR | 差异 |
|---|---|---|---|
| 初始化耗时 | 1.01 秒 | 2.42 秒 | PaddleOCR慢约1.4秒 |
| 单张推理耗时 | 1.53 秒 | 15.74 秒 | PaddleOCR慢约10.3倍（CPU模式） |
| 识别文字数量 | 82 处 | 116 处 | PaddleOCR多识别41% |
| 平均置信度 | 0.515 | 0.845 | PaddleOCR置信度高出约64% |

**结论**：PaddleOCR在识别数量与置信度上明显优于EasyOCR（多识别41%文字、置信度高64%），但在当前CPU推理环境下速度慢约10倍。这是符合预期的精度-速度权衡——PaddleOCR的检测模型更强但计算量更大，若后续接入GPU加速，预计速度差距会显著缩小。第3-4周搭建端到端流程时，可视实际任务对实时性的要求，在两者间灵活切换或考虑并行部署GPU版PaddleOCR。

### 2.3 鼠标键盘控制模块（control/mouse_keyboard.py）

基于pyautogui实现点击、双击、移动、拖拽、滚动、文本输入（含pyperclip中转方案支持中文）、组合键等操作接口。鼠标定位与移动的安全测试（无副作用）实测坐标误差在5像素以内。进一步在记事本窗口中完成了真实点击与文本输入的端到端测试：点击准确命中目标窗口，输入文本成功写入，验证了"感知坐标→控制执行"这一链路的可用性。拖拽与滚动操作代码已实现但尚未实测，留待第3-4周端到端流程集成时一并验证。

## 三、环境与依赖问题排查记录

PaddleOCR集成过程中连续遇到四类问题，记录如下，为后续同类环境搭建提供参考：

| 问题现象 | 原因 | 解决方案 |
|---|---|---|
| PaddleOCR依赖包缺失，提示paddle_static不可用 | paddleocr包只是上层封装，底层推理引擎paddlepaddle需单独安装 | `pip install paddlepaddle` |
| 模型下载404 / 连接不上模型托管服务 | PaddleOCR 3.x默认额外加载文档方向分类、图像矫正等预处理模型，触发多次联网下载校验 | 关闭非必要预处理：`use_doc_orientation_classify=False, use_doc_unwarping=False, use_textline_orientation=False`，仅保留核心检测识别模型 |
| CPU推理报错 ConvertPirAttribute2RuntimeAttribute not support（oneDNN算子不兼容） | 当前paddlepaddle版本的oneDNN后端与所用模型的PIR格式存在兼容性问题 | 初始化时增加 `enable_mkldnn=False`，关闭oneDNN加速，改用基础CPU执行路径 |
| 解析OCR结果报错 too many values to unpack | PaddleOCR 3.x已将.ocr()标记为废弃，返回数据结构与旧版本不同 | 改用官方推荐的`.predict()`接口，按新版字段(rec_texts/rec_scores/rec_boxes)重新解析 |
| 自动读取最新截图误选到标注图/区域图 | 文件查找按修改时间取最新，多个脚本连续运行时容易选中非预期的图片 | 已知限制，建议测试前先单独跑一次全屏截图，确保用于OCR对比的图片是原始未标注的全屏图 |
